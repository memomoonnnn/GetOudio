# 授权/关于

「Get Oudio」是为音频工作者们开发的实用性工具。软件做了一些微小的工作，集成了一些优秀的开源项目，并使它们可以在MacOS上被方便地调用，感谢这些开发者的无私奉献！

过去类似 Permute、Downie 这样的软件，你需要先打开窗口、拖放文件、设定参数再启动进程，完成任务后还要手动退出，这种交互设计在我看来非常愚蠢。于是 Get Oudio 参考了一众解压缩软件的设计逻辑，将操作入口全集成在 macOS 本身的组件中；所有功能预制化，不需要任何临时的额外设置；全程也没有任何窗口干扰，只有右上角的横幅通知告知你进程状态，就好像 Mac 天生就自带这个功能一般。

![300](attachments/Get%20Oudio%20Doc/file-20260704034608541.webp)

关于各个功能的使用教程，请切换左侧边栏，在不同的设置页中一一查看，或在Github上查看简易的视频演示。

另外，这个设置窗口的背景是透明的。系统使用浅色模式时，请不要把窗口放在纯黑色背景上；使用深色模式时，请不要放在纯白色背景上，这会使GUI变得难以辨别。

![|500](attachments/Get%20Oudio%20Doc/file-20260713123652058.webp)

# 音频重编码

依靠[FFmpeg](https://github.com/FFmpeg/FFmpeg)，现在仅需在访达中右键支持的音频文件（支持多个），并选择Get Oudio子菜单中的预制，即可重编码到当前目录：

![300](attachments/Get%20Oudio%20Doc/file-20260704034734987.webp)

如果是视频文件，则可以提取其音频轨，遵循原始格式：

![300](attachments/Get%20Oudio%20Doc/file-20260704034835540.webp)

你还可以将特定音频文件的默认打开方式设为「Get Oudio」，这样当你双击它们，光标位置就会浮现目标编码列表，选择后即可开始进程。

![200](attachments/Get%20Oudio%20Doc/file-20260704035014628.webp)

  

一定程度上，这并不影响你播放并聆听音频，因为QuickTime的空格预览仍然生效，这操作远比双击打开一个新的QuickTime窗口要常用得多：

  

![300](attachments/Get%20Oudio%20Doc/file-20260704035158344.webp)

  

受支持的编码包括AAC、MP3、Vorbis、Opus、ALAC/FLAC和PCM。需要说明：

  

- AAC和MP3的输出预设只包含不同的码率。即便它们本来支持不同的采样率，但我们往往不会在意它；而且DAW在导入这类有损压缩音频时，会自动解码成当前工程采样率的PCM码流，所以它们的采样率并不会带来不便。

- Vorbis使用ogg封装。由于Vorbis设计上倾向于可变码率（VBR），因此预设围绕q值展开。

- Opus遵循官方规范使用ogg封装，但文件后缀使用opus。它的设计同样倾向可变码率，而且由于支持多声道，预设指定的是每个声道单独的目标码率——如128kbps，对立体声音轨来说合并后码率等效为256kbps。另外，由于Opus的压缩质量很高，更高码率的Opus已经背离了有损压缩的初衷，边际收益过低，所以不加预设。

- AAC与ALAC遵循Apple规范，均使用m4a封装。

- PCM的输出提供wav和使用率相对较低的aiff。aiff相比于wav的优势是可以存储元数据，其余并无差异。

- 额外添加了caf作为支持的输入格式。caf可以封装任意编码、无限长度、无限大小的音频流，甚至可以嵌入MIDI数据、Warp标记、波形概览和SMPTE。这种丰富度使得将其作为输出缺乏目的性；而且由于只受苹果生态支持，几乎不会出现在DAW之外需要特地转换为caf的场景。故仅支持作为输入。

- 不同编码之间的转换，就原理而言不存在优劣之分，因为算法和结构是编码格式和封装容器本身决定的；可当涉及采样率的转换，本软件使用的FFmpeg算法是否劣于各大DAW还有待商榷，若有洁癖的话不建议使用。

# Apple Music 下载

按上方流程完成依赖安装与初始化，需要说明：

- 完成依赖安装和初始化后，「Get Oudio」将不可避免地在你的电脑上占用超过1GB的空间，这是因为「Get Oudio」需要为Wrapper搭一个虚拟机，目前采用的Colima+Docker CLI已经是体积最小的方案了。

- 访达中可能会显示超过6GB，这是虚拟机的逻辑大小，并不是实际占用的空间，大可放心。

- 源项目也包含不依赖Wrapper的路径，即以浏览器身份爬取数据，空间占用仅有本体的几MB，但这种方案只能获得128kbps的AAC文件，我相信应该没有人需要它，所以也懒得做进来了...

- 依赖项从官方地址安装，期间需要开启科学上网的TUN（虚拟网卡）模式。各依赖项不支持寻找本地已有的复用，因为倘若你电脑上已经有Docker，那么我只能假定你有足够的动手能力，于是更加建议你获取源项目并搭建Shortcuts的自动化流程，这比该软件更稳定也更易于维护，与Mac的集成也更加舒适，可以[在这里参考我搭建的流程](https://www.icloud.com/shortcuts/f8ec294188db497fa9e6a2cf023a1d44)。

- 填写帐密并通过登录验证是必要的，「Get Oudio」不会私自获取你的用户信息。

依靠[Apple-Music-Downloader](https://github.com/zhaarey/apple-music-downloader)，现在可以在Apple Music中分享任意歌曲、播放列表或专辑到「Get Oudio」以下载到目标文件夹，支持ALAC、AAC（256kbps）与Atoms。

![500](attachments/Get%20Oudio%20Doc/file-20260704055235156.webp)

下载进程在选择横幅通知的格式Action后即刻开始：

![300](attachments/Get%20Oudio%20Doc/file-20260704055429493.webp)

当然你也可以固定每次下载的格式，这样就不需要多点一次按钮了。

下载时，「Get Oudio」还会每30秒发出横幅通知指示当前的进度，若发现长时间没有反应可以在上方急停。

指定的输出目录下，歌曲将以「歌手-专辑」的层级进行分类，同时保存专辑封面：

![|200](attachments/Get%20Oudio%20Doc/file-20260704055718733.webp)

# 录音

在右侧的通知中心或者桌面添加「Get Oudio」的小组件，即可通过它启动录音进程：

![|400](attachments/Get%20Oudio%20Doc/file-20260713114729053.webp)

该小组件实则是一个后台应用唤醒器，因此当你点击开始后，通知中心会立刻收回。虽然很不方便，但也是无奈之举，避免后台一直有个Agent程序吃内存。

录音开始后，「Get Oudio」会自动切换系统的声音播放设备为你指定的Protools AudioBridge，持续捕获其信号，同时发送到切换前的设备上， 以便你继续监听。

值得注意的是，Get Oudio会排除系统提示音，你不必担心录音收入恼人的通知声——这受益于macOS提供的官方接口。

小组件样式也会发生变化，以指示录音状态。再次点击即结束录音，播放设备也将自动恢复为先前状态：

![|200](attachments/Get%20Oudio%20Doc/file-20260713120152890.webp)

「Get Oudio」默认对音频进行后处理，标准化并去除头尾的无声片段，同时将最终文件复制到你的剪贴板，使你能快速Paste到任何位置。

额外说明：

- 当前暂时仅支持Protools AudioBridge的2-A和2-B。就原理来说，该录音功能可以支持其他设备，如Loopback、Blackhole或者声卡自带的Loopback通道。但这样会面临复杂的通道映射问题，兴许在将来会支持。如今选择2-A和2-B主要是因为他们足够常见、稳定。
- 采样率跟随AudioBridge的采样率，输出为24bit。
- 你可以选择将录音文件保存到指定的目录以方便管理。
- 你可以选择性启用后处理功能。标准化采用Peak模式，限制到-0.1dBFS；去除无声则可以调节阈值和额外垫付的时长。

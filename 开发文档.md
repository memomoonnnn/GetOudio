# 概述

「Get Oudio」是一款Macos原生的软件，用于执行一些与音频转换有关的工作。

本质上，「Get Oudio」是一个脚本执行器，依托开源项目ffmpeg、[ncmdump](https://github.com/taurusxin/ncmdump)和[apple-music-downloader](https://github.com/zhaarey/apple-music-downloader)，让这些工具可以被方便地调用。

功能上，分为四个板块：

# 重编码音频

参照此前使用的脚本上：

```
#!/bin/zsh

FFMPEG_BIN="/opt/homebrew/bin/ffmpeg"
SUCCESS_COUNT=0
FAILURE_COUNT=0

for f in "$@"
do
    if [[ -f "$f" ]]; then

        DIR_NAME=$(dirname "$f")
        BASE_NAME=$(basename "$f")
        
        BASE_NAME_WITHOUT_EXT="${BASE_NAME:r}"
        
        OUTPUT_FILE="${DIR_NAME}/${BASE_NAME_WITHOUT_EXT}.m4a"
        
        "$FFMPEG_BIN" -i "$f" -acodec aac -b:a 320k -map_metadata 0 -y -vn "$OUTPUT_FILE"
        
        if [ $? -eq 0 ]; then
            SUCCESS_COUNT=$((SUCCESS_COUNT + 1))
        else
            FAILURE_COUNT=$((FAILURE_COUNT + 1))
        fi
    fi
done

TOTAL_PROCESSED=$((SUCCESS_COUNT + FAILURE_COUNT))

if [ $TOTAL_PROCESSED -eq 0 ]; then
    TITLE="没有文件被处理"
    MESSAGE="请确保您选择了有效的文件。"
elif [ $FAILURE_COUNT -eq 0 ]; then
    TITLE="✅ 所有文件转换完成"
    MESSAGE="成功转换 ${SUCCESS_COUNT} 个文件为 AAC。"
else
    TITLE="⚠️ 转换完成，但有错误"
    MESSAGE="总共处理 ${TOTAL_PROCESSED} 个文件。成功 ${SUCCESS_COUNT} 个，失败 ${FAILURE_COUNT} 个 (可能是非音频文件)。"
fi

/usr/bin/osascript -e "display notification \"$MESSAGE\" with title \"$TITLE\""

exit 0
```

该脚本使用ffmpeg重编码输入的音频文件为320Kbps的AAC文件，并用`.m4a`封装。

「Get Oudio」的目的是添加访达的右键拓展，使在访达中右键音频文件时有「Get Oudio」的选单，可以选择多个预制，重编码并封装音频输出到当前目录下，需要的预制有：

- AAC编码、m4a封装。128Kbps、256Kbps、320Kbps各一个；
- mp3编码、mp3封装。128Kbps、256Kbps、320Kbps各一个；
- ALAC编码、m4a封装。24bit 48KHz、16bit 48KHz规格与音源原始规格各一个。
- FLAC编码、flac封装。24bit 48KHz、16bit 48KHz规格与音源原始规格各一个。
- PCM编码、wav封装。24bit 48KHz、16bit 48KHz规格与音源原始规格各一个。

# 从视频中提取音频

同样参照先前使用的脚本：

```
#!/bin/zsh

FFMPEG_BIN="/opt/homebrew/bin/ffmpeg"
SUCCESS_COUNT=0
FAILURE_COUNT=0

for f in "$@"
do
    if [[ -f "$f" ]]; then

        DIR_NAME=$(dirname "$f")
        BASE_NAME=$(basename "$f")
        
        BASE_NAME_WITHOUT_EXT="${BASE_NAME%.*}"  # 使用更通用的删除扩展名方法
        
        # 获取音频编码格式
        AUDIO_CODEC=$("$FFMPEG_BIN" -i "$f" 2>&1 | grep -o 'Audio: \([^,]*\)' | head -n 1 | awk -F ' ' '{print $2}')
        
        if [[ -z "$AUDIO_CODEC" ]]; then
            echo "未能识别音频编码格式：$f"
            FAILURE_COUNT=$((FAILURE_COUNT + 1))
            continue
        fi

        # 根据音频编码格式选择合适的容器格式
        case "$AUDIO_CODEC" in
            "aac")
                OUTPUT_FILE="${DIR_NAME}/${BASE_NAME_WITHOUT_EXT}.m4a"
                ;;
            "vorbis")
                OUTPUT_FILE="${DIR_NAME}/${BASE_NAME_WITHOUT_EXT}.ogg"
                ;;
            "mp3")
                OUTPUT_FILE="${DIR_NAME}/${BASE_NAME_WITHOUT_EXT}.mp3"
                ;;
            "flac")
                OUTPUT_FILE="${DIR_NAME}/${BASE_NAME_WITHOUT_EXT}.flac"
                ;;
            "opus")
                OUTPUT_FILE="${DIR_NAME}/${BASE_NAME_WITHOUT_EXT}.ogg"
                ;;
            "ac-3")
                OUTPUT_FILE="${DIR_NAME}/${BASE_NAME_WITHOUT_EXT}.ac3"
                ;;
            *)
                OUTPUT_FILE="${DIR_NAME}/${BASE_NAME_WITHOUT_EXT}_audio.${AUDIO_CODEC}"
                ;;
        esac

        # 提取音频轨道并保持编码不变，封装成对应容器格式
        "$FFMPEG_BIN" -i "$f" -c:a copy -map_metadata 0 -vn "$OUTPUT_FILE"
        
        if [ $? -eq 0 ]; then
            SUCCESS_COUNT=$((SUCCESS_COUNT + 1))
        else
            FAILURE_COUNT=$((FAILURE_COUNT + 1))
        fi
    fi
done

TOTAL_PROCESSED=$((SUCCESS_COUNT + FAILURE_COUNT))

if [ $TOTAL_PROCESSED -eq 0 ]; then
    TITLE="没有文件被处理"
    MESSAGE="请确保您选择了有效的视频文件。"
elif [ $FAILURE_COUNT -eq 0 ]; then
    TITLE="✅ 所有文件转换完成"
    MESSAGE="成功提取并重新封装音频轨道 ${SUCCESS_COUNT} 个视频。"
else
    TITLE="⚠️ 转换完成，但有错误"
    MESSAGE="总共处理 ${TOTAL_PROCESSED} 个文件。成功 ${SUCCESS_COUNT} 个，失败 ${FAILURE_COUNT} 个 (可能是非视频文件)。"
fi

/usr/bin/osascript -e "display notification \"$MESSAGE\" with title \"$TITLE\""

exit 0
```

同样使用ffmpeg，按照视频中音频轨的原始格式将其剥离出来。

# NCM格式转换

需要项目[ncmdump](https://github.com/taurusxin/ncmdump)，参照之前使用的脚本：

```
# 定义变量
TOOL_PATH="/opt/homebrew/bin/ncmdump"
OUTPUT_DIR="/Users/shengjiacheng/Desktop/Music"

# 1. 确保输出目录存在，不存在则创建
if [ ! -d "$OUTPUT_DIR" ]; then
    mkdir -p "$OUTPUT_DIR"
fi

# 2. 执行转换命令
# "$@" 代表用户选中的所有文件列表
# 2>&1 将错误输出也捕获，以便调试
ERROR_LOG=$("$TOOL_PATH" -o "$OUTPUT_DIR" "$@" 2>&1)
EXIT_CODE=$?

# 3. 根据执行结果发送 macOS 通知
if [ $EXIT_CODE -eq 0 ]; then
    # 获取文件数量
    FILE_COUNT=$#
    
    # 调用 AppleScript 发送成功通知
    osascript -e "display notification \"已成功转换 $FILE_COUNT 个文件至桌面 Music 目录\" with title \"ncmdump 转换完成\" sound name \"Glass\""
    
    # 将结果输出给 Automator (可选)
    echo "成功转换 $FILE_COUNT 个文件"
else
    # 调用 AppleScript 发送失败通知
    osascript -e "display notification \"转换失败，请检查文件或路径\" with title \"ncmdump 错误\" sound name \"Basso\""
    
    # 如果失败，输出错误日志
    echo "错误: $ERROR_LOG"
fi
```

使用ncmdump的原始命令。

「Get Oudio」需要能够绑定`.ncm`格式的文件，使用户在用此打开方式双击ncm文件时就触发ncmdump的脚本执行。

# 从Apple Music下载

需要项目[apple-music-downloader](https://github.com/zhaarey/apple-music-downloader)，它依赖go、gpac、Docker CLI和Colima。为避免 Docker Desktop 这类桌面软件常驻和弹窗，「Get Oudio」只依赖命令行版 Docker 与 Colima 提供的本地后台 engine，作者 wrapper 镜像仍按原流程拉取和运行。

参照先前使用的快捷指令：https://www.icloud.com/shortcuts/e8e517722efb4189b04b00108914cdfe

「Get Oudio」需要添加一个用于“共享”的访达拓展，使其在Apple Music中菜单选择“共享”时，可以传送Url到「Get Oudio」里，然后软件将根据传入的Url执行下载程序。

# 图形化界面

图形化界面，使用Swift UI，遵循Apple开发风格与规范。由于「Get Oudio」的主要功能集中在对右键菜单的拓展中，所以图形化界面只有两个板块：设置和「apple-music-downloader」的初始化。

## 设置项

需要的设置有：

- 重编码音频项目的开关。这使用户能够按个人喜好决定开启重编码功能里的哪些。
- NCM转换的输出选项，用户可以决定NCM转换后输出到原目录下，也可以选择输出到指定目录下。
- Apple Music下载的选项：
	- 用户可以决定在Apple Music中共享到「Get Oudio」后触发的下载程序，文件输出到哪个目录；
	- 用户可以决定，每次下载是询问下载的类型（ALAC还是AAC还是Atoms），还是只默认下载该选项指定的类型。
  
## Apple Music下载器的初始化

初始化时，模拟正常的终端操作：

1. cd进安装目录，执行`docker --context colima run --platform linux/amd64 -v ./rootfs/data:/app/rootfs/data -e args="-L username:password -F" --rm ghcr.io/itouakirai/wrapper:x86`，将其中的`username`和`password`字段换成用户自己有订阅权限的帐密；
2. 随后Apple将向用户发送验证码，用户需要在另一个终端里cd进安装目录，然后echo验证码到`./rootfs/data/`下的`2fa.txt`里；
3. 直到终端返回`type 6`，即初始化成功。

「Get Oudio」需要让这个过程GUI化，用户只需要在提示框里输入用户名和密码，输入验证码即可完成初始化。

# 依赖处理

「Get Oudio」依赖于这些优秀的开源项目，所以需要保证这些库本体相互独立，并且可以获得来自这些库的更新。

同时，需要封装所有的依赖状态和安装流程。ncmdump 与 apple-music-downloader 作为「Get Oudio」私有组件嵌入 App bundle，不要求用户全局安装；apple-music-downloader 的运行时配置由 App 从内置模板生成到 App Group 工作目录，并把下载保存目录指向用户设置项。wrapper 不作为二进制文件嵌入 App，而是由「Get Oudio」在用户触发 Apple Music 功能时后台执行`colima start --runtime docker`，随后用`docker --context colima`拉取和运行镜像，让用户无需手动打开 Docker Desktop 或维护全局散落文件。

「Get-Oudio」所有脚本运行时均在后台静默，只在完成或报错时发送横幅通知。
